5. Onboarding API Route

File: app/api/onboarding/route.ts

  • POST endpoint — no AI, pure rules engine only
  • Validates: citizenship (SC/PR/Foreigner), age (integer 0–120), monthly_pchi (≥0), annual_value (≥0)
  • Invalid JSON → 400; failed validation → 400 with specific field error
  • All optional fields (medisave_balance, adl_needs_assistance, etc.) spread conditionally
  • Returns: { schemes: SchemeMatch[], profile: UserProfile }
